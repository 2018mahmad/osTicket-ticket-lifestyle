This project implements a structured workflow that improves:

- Response time  
- Ticket visibility  
- Accountability  
- Customer satisfaction

## ‚è±Ô∏è SLA & Priority Demonstration

Service Level Agreements (SLAs) are used to prioritize tickets based on severity:

- **Sev-A (1 hour, 24/7):** Critical system outages  
- **Sev-B (4 hours, 24/7):** High-priority software or business-impacting issues  
- **Sev-C (2 hours, business hours):** Low-priority issues such as slow performance or minor requests  

This ensures critical incidents are addressed immediately while lower-priority issues are handled efficiently.

## üîß Triage & Escalation

- Tickets are evaluated for severity and impact  
- Low-level issues handled by Level I support  
- Complex issues escalated to Level II or System Administrators  
- Communication maintained with users throughout the process
